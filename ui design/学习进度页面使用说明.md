# 学习进度页面设计稿使用说明

## 📋 页面概览

学习进度页面展示用户的练习数据、统计图表和成就系统，帮助用户跟踪学习表现。

## 🎨 页面结构

### 1. 页面标题区域
- **主标题**：学习进度（24px，Medium）
- **副标题**：跟踪你的学习表现（14px，Regular）

### 2. 本周概览卡片（紫色渐变）
- **背景**：Indigo 600 → Purple 600 渐变
- **内容**：2x2 网格显示四个关键指标
  - 平均准确率：84%
  - 练习次数：28
  - 学习时长：3.5h
  - 连续天数：7天

### 3. 每日得分图表
- **类型**：条形图（Bar Chart）
- **数据**：周一到周日的每日得分
- **样例数据**：45, 62, 58, 73, 85, 78, 90
- **样式**：Indigo 600 填充，圆角柱状图

### 4. 成就激励卡片网格
- **布局**：2列网格，每行2个成就卡片
- **成就类型**：
  - 🎯 首次练习（已激活，黄色背景）
  - 🔥 连续7天（已激活，黄色背景）
  - ⭐ 准确率90%（已激活，黄色背景）
  - 🏆 百发百中（未激活，灰色半透明）

### 5. 下一个目标卡片
- **背景**：绿色浅色背景（#DCFCE7）
- **内容**：显示距离下一个成就的进度
- **示例**：再练习 23 次即可解锁「练习达人」徽章！

### 6. 底部导航栏
- **项目**：首页、进度（激活）、设置
- **样式**：白色背景，顶部边框
- **激活状态**：Indigo 600 颜色

## 🚀 使用方法

### 在 Figma 中使用

1. 打开 Figma 桌面应用
2. 进入你的设计文件
3. 打开插件面板：`Plugins` → `Development` → `Import plugin from manifest`
4. 选择 `ui design/create-progress-page.js` 文件
5. 运行插件
6. 点击"创建设计稿"按钮

### 自定义数据

如果需要修改页面数据，可以编辑 `create-progress-page.js` 文件中的以下变量：

#### 统计数据（本周概览）
```javascript
const stats = [
  { label: '平均准确率', value: '84%', x: 0, y: 0 },
  { label: '练习次数', value: '28', x: 1, y: 0 },
  { label: '学习时长', value: '3.5h', x: 0, y: 1 },
  { label: '连续天数', value: '7天', x: 1, y: 1 },
];
```

#### 每日得分数据
```javascript
const dailyScores = [
  { day: '周一', score: 45, value: 45 },
  { day: '周二', score: 62, value: 62 },
  { day: '周三', score: 58, value: 58 },
  { day: '周四', score: 73, value: 73 },
  { day: '周五', score: 85, value: 85 },
  { day: '周六', score: 78, value: 78 },
  { day: '周日', score: 90, value: 90 },

```

#### 成就数据
```javascript
const achievements = [
  { icon: '🎯', title: '首次练习', desc: '完成第一次识谱练习', active: true, x: 0 },

  { icon: '⭐', title: '准确率90%', desc: '单次练习准确率达90%', active: true, x: 0 },
```

## 🎨 设计规范

本页面完全遵循 `ui设计稿生成规范.md` 中的设计标准：

### 颜色使用
- ✅ SOLID 类型颜色只包含 `{ r, g, b }`
- ✅ 渐变颜色包含 `{ r, g, b, a: 1 }`
- ✅ 渐变包含 `gradientTransform`

### 文字创建
- ✅ 先加载字体 → 先设置 fontName → 设置 fontSize → 最后设置 characters
- ✅ 使用 Inter 字体，支持 Regular 和 Medium 字重

### 错误处理
- ✅ 使用 try-catch 包裹所有代码
- ✅ 提供友好的错误提示
- ✅ 记录详细错误信息到控制台

### 代码组织
- ✅ 所有代码内联，不使用 require/import
- ✅ 颜色、字体等设计令牌直接定义在文件中

## 📐 尺寸规格

- **屏幕尺寸**：375 x 812px（移动端标准尺寸）
- **页面边距**：24px
- **卡片圆角**：
  - 主要卡片：16px
  - 成就卡片：12px
- **卡片间距**：24px（垂直），12px（水平网格）
- **字体大小**：
  - 页面标题：24px
  - 区块标题：16px
  - 数据数值：24px
  - 正文：14px
  - 小字：12px

## 🎯 设计亮点

1. **视觉层次清晰**：使用渐变卡片突出本周概览，白色卡片承载详细信息
2. **数据可视化**：条形图直观展示每日得分趋势
3. **激励机制**：成就系统激励用户持续练习
4. **状态反馈**：已激活/未激活成就的视觉区分
5. **行动引导**：下一个目标卡片引导用户继续努力

## 📝 注意事项

1. 本设计稿使用 Emoji 作为图标占位符，实际开发中建议使用图标库（如 Lucide React）
2. 图表数据为示例数据，实际应用中需要连接真实的数据源
3. 页面高度可能超过一屏，实际应用中需要支持滚动
4. 底部导航栏固定在屏幕底部
